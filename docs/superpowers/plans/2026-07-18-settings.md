# Claude Deck — In-App Settings Page Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox steps.

**Goal:** A `⚙ Settings` screen (opened from a bottom-of-sidebar entry, `Ctrl-a s`, or a click) to manage notifications and other options, persisted to a cross-platform config file so choices survive restarts.

**Architecture:** A `config.rs` module (serde struct ↔ `dirs::config_dir()/claude-deck/config.json`). On startup: load config → set the corresponding `App` fields; existing CLI flags then override for that run. A new `Focus::Settings` renders a navigable settings list in the main pane; toggling/editing an item updates the `App` field AND saves the config immediately. Sidebar gets a `⚙ Settings` row pinned at the bottom.

## Global Constraints
- **Cross-platform** config path via `dirs`; no `std::os::unix`-only code.
- **Config is the persisted store; CLI flags override at launch** (flags win for that run, don't persist unless changed in the UI).
- Never call the Anthropic API / handle tokens.
- Settings screen takes no PTY input (like Home); keystrokes drive the settings navigator.

## Settings managed
`dnd` (mute ALL alerts — bell + desktop + ntfy), `bell`, `desktop_notify`, `ntfy_topic` (string), `reap_idle` (bool), `reap_timeout_secs` (u64), `mem_warn_mb` (u64), `nerd_icons` (bool).

### Task SET1: Config persistence + Settings focus + sidebar entry + read/navigate + toggles

**Files:** `Cargo.toml` (dirs already added), `src/config.rs` (new), `src/app.rs`, `src/ui.rs`

**Interfaces:**
- `src/config.rs` (`mod config;`): `#[derive(Serialize, Deserialize, Clone, Default)] struct Config { dnd, bell, desktop_notify, ntfy_topic: Option<String>, reap_idle, reap_timeout_secs, mem_warn_mb, nerd_icons }` with sensible `Default` (bell=true, desktop_notify=true, reap_idle=false, reap_timeout_secs=600, mem_warn_mb=4096, nerd_icons=false, dnd=false, ntfy_topic=None). `path()` via `dirs::config_dir()`. `load()`/`save(&Config)` (+ testable `load_from`/`save_to`). Round-trip unit test.
- `App`: `enum Focus` gains `Settings`. `App.config: Config` + `App.settings_cursor: usize`. On `App::new`: `config = config::load()`; apply it to fields (bell_on, notify_on, ntfy_topic, reap_idle, reap_timeout, mem_warn_kb, icon mode); THEN apply CLI flags (override). A `dnd` field: when true, bell/desktop/ntfy are all suppressed at fire-time (guard the notification branch with `!self.config.dnd`).
- Sidebar (`ui.rs`): render a `⚙ Settings` row pinned at the BOTTOM of the sidebar (after the session list, e.g. rendered at the last line), highlighted when `focus == Settings`. `Ctrl-a s` and a mouse click on that row set `focus = Focus::Settings` (mouse hit-testing: the bottom row maps to Settings).
- Settings view (`ui.rs`): when `focus == Settings`, main pane shows the settings list — one row per setting: `label ......... value` (value = `on/off` for bools, the topic/number for others), with the `settings_cursor` row highlighted. Header "Settings — ↑/↓ move · Enter/Space toggle · Esc back".
- Keys while `focus == Settings`: ↑/↓ (or k/j) move `settings_cursor`; Enter/Space toggles the boolean setting at the cursor (updates the App field + `self.config` + `config::save`); Esc → `focus = Home`. (Text/number editing is SET2 — for SET1, Enter on `ntfy_topic`/numbers can be a no-op or cycle; booleans fully work.)

- [ ] Step 2: `Focus::Settings`, config load+apply on startup (flags override), sidebar `⚙ Settings` row + `Ctrl-a s` + click, settings render + cursor nav + boolean toggles + save + `dnd` suppression guard.
