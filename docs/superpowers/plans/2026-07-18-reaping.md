# Claude Deck — Cross-Platform Pass + Memory Guardian (RAM) Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** (1) Make claude-deck run on Windows/macOS/Linux (today the hooks IPC + notifications + claude-path are macOS/Unix-only). (2) Stop `claude` from eating all RAM (user saw ~43 GB freeze the machine) via a **memory guardian**: monitor each session's memory and, over a cap, kill + `--resume` it — RAM resets, conversation preserved. Plus idle-park/revive + pin.

**Portability strategy:** use standard cross-platform crates — **TCP localhost** for hook IPC (std only), **`notify-rust`** for notifications, **`sysinfo`** for memory, **`which`** for the binary lookup. `portable-pty`/`crossterm`/`ratatui` are already portable. A **GitHub Actions CI** builds + tests on ubuntu/macos/windows (the dev machine is macOS-only, so CI is the cross-platform guarantee).

**Verified against real `claude`:** `--resume <id>` restores the conversation AND the resumed process's hooks report the **same `session_id`** (state tracking survives a restart). `Child::process_id()` gives the pid to measure.

## Global Constraints

- **No platform lock-in:** no `std::os::unix`-only code paths without a portable equivalent; must compile on Windows/macOS/Linux. Prefer portable crates over per-OS shell commands.
- **Never call the Anthropic API / handle tokens.** Restart/revive is `claude --resume`, the real CLI.
- **Freeing RAM requires killing the `claude` process** (`killer.kill()` + drop `PtySession`); restart respawns immediately via `--resume`.
- **Hard cap protects the machine** — overrides pin/focus/running (a runaway must never freeze the PC). Soft-cap/idle park respects pin + never touches the focused or non-`Idle` session.
- **Disableable:** `--no-reap`, `--reap-timeout 0`, `--mem-soft 0`, `--mem-hard 0`.

## Config (flags, defaults)
`--reap-timeout <secs>` (300; 0 off) · `--mem-soft <MB>` (4096; 0 off) · `--mem-hard <MB>` (12288; 0 off) · `--no-reap`.

### Task C1: Portable hook IPC — TCP localhost (replaces Unix socket)

**Files:** `src/hooks.rs`, `src/lib.rs`, `src/app.rs`

The current `hooks.rs` uses `std::os::unix::net::{UnixListener,UnixStream}` — Windows can't compile it. Replace with a `127.0.0.1` TCP socket.

**Interfaces:**
- `hooks::listen(on_event) -> io::Result<u16>` — bind `TcpListener::bind("127.0.0.1:0")`, return the OS-assigned port; spawn the accept loop (per-connection worker thread, from the prior fix). No socket file.
- `hooks::forward(port_or_addr: &str)` — connect `TcpStream::connect(("127.0.0.1", port))`, write stdin, done. The `__hook` arg is now the port (string).
- `hooks::settings_json(binary, port)` — hook command becomes `"<abs-binary> __hook <port>"`.
- `App`: no more socket path / socket-file cleanup; keep the settings-file path + cleanup. Store the `port`; write the settings file using it.

### Task R2: Memory guardian (sysinfo) + idle timer + park/restart

**Files:** `src/app.rs`, `src/pty.rs` (expose pid), `src/mem.rs` (new, sysinfo-based)

- `PtySession` stores the child pid (`child.process_id()` captured before the child moves to the waiter thread).
- `src/mem.rs`: hold a `sysinfo::System`; `rss_kb(&mut self, pid) -> Option<u64>` refreshes that process and returns its memory (KB); `total_ram_kb()` for defaults. (sysinfo is cross-platform.)
- `AppEvent::Tick` (timer thread, ~10s). `App` fields: `reap_timeout/mem_soft_kb/mem_hard_kb` (flags), `idle_since: HashMap<String,Instant>`, `rss: HashMap<String,u64>`.
- Pure helpers (unit-test): `should_restart(rss, hard)`; `should_park(state,focused,pinned,idle_for,rss,timeout,soft)`.
- On `Tick`: measure each live session's rss (store); apply `should_restart` first → `restart_session` (kill + respawn `--resume`, stay live); else `should_park` → `park_session` (kill + `None` + `Parked`).

## Self-Review

**Coverage:** portable IPC (C1), portable notify/claude-path/deps (C2), CI on 3 OSes (C3), anti-freeze hard-cap restart (R2 — the 43 GB problem), soft/idle park (R2), pin (R1), revive (R3), RSS in sidebar (R3), real-claude verify incl. RAM-reclaimed + session_id continuity + no leaked children (R4). ✓

**Risks:**
- **Can't test Win/Linux locally** → CI (C3) is the net; keep code portable-by-construction (no `std::os::unix` without cfg).
- **`notify-rust` on Linux** needs libdbus at build (CI installs it) + a notification daemon at runtime (degrades gracefully — errors ignored).
- **Hard-cap on an active session** loses the in-flight reply; high default (12 GB) + configurable.
- **`Option<PtySession>` refactor (R1)** riskiest; compile-driven.
- **pid capture** before the child moves to the waiter thread; `rss` None on failure → session simply not acted on.
