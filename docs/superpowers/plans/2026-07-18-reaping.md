# Claude Deck — Cross-Platform Pass + Memory Guardian (RAM) Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** (1) Make claude-deck run on Windows/macOS/Linux (hooks IPC + notifications + claude-path are currently Unix-only). (2) Help the user avoid the RAM freeze they hit (~43 GB) **without ever auto-killing a working session** — via an *advisory* memory guardian: show each session's RAM, warn early, and let the user restart heavy sessions themselves (with confirmation; `--resume` preserves the conversation). Opt-in idle-parking + pin for those who want automatic RAM savings.

**SAFETY (non-negotiable):** claude-deck **never automatically terminates or restarts an ACTIVE session** (Running / WaitingOnYou / Starting). An active session may be running an agent or background subagents — killing it loses that work. Automatic process-killing is limited to **opt-in** idle-parking, which only ever touches a **truly idle** session (its turn finished via the `Stop`/`idle_prompt` hook), is unfocused, unpinned, and timed-out. Any user-triggered restart of a live session asks for confirmation first.

**Portability strategy:** TCP localhost for hook IPC (std only) · `notify-rust` notifications · `sysinfo` memory · `which` binary lookup. `portable-pty`/`crossterm`/`ratatui` already portable. GitHub Actions CI builds+tests on ubuntu/macos/windows (dev machine is macOS-only).

**Verified against real `claude`:** `--resume <id>` restores the conversation AND resumed hooks report the same `session_id`. `Child::process_id()` gives the pid to measure.

## Config (flags, defaults)
- `--mem-warn <MB>` sidebar/notification warning threshold, default `4096`; `0` disables warnings.
- `--reap-idle` **opt-in** automatic idle-parking (default OFF — nothing auto-parks unless set).
- `--reap-timeout <secs>` idle-park timeout when `--reap-idle` is on, default `600`.
- No `--mem-hard`. There is NO automatic restart of active sessions.

## Global Constraints
- **Never auto-terminate an active session** (see SAFETY). No memory threshold triggers an automatic kill/restart of a Running/WaitingOnYou/Starting session — only a warning.
- **Idle-parking is opt-in and idle-only**: requires `--reap-idle`, and parks only Idle + unfocused + unpinned + timed-out sessions.
- **User-triggered restart requires confirmation** (a y/n prompt naming the session and warning in-flight work is lost).
- **No platform lock-in:** no `std::os::unix`-only paths without a portable equivalent; must compile on Windows/macOS/Linux.
- **Never call the Anthropic API / handle tokens.** Restart/revive is `claude --resume`.
- **Freeing RAM = killing the `claude` process** (`killer.kill()` + drop `PtySession`); revive/restart respawns via `--resume`.

### Task R2: Memory monitor + warnings + opt-in idle park (NO auto-restart)

**Files:** `src/app.rs`, `src/pty.rs` (expose pid), `src/mem.rs` (new, sysinfo)

- `PtySession` stores the child pid (`child.process_id()`, captured before the child moves to the waiter thread).
- `src/mem.rs`: a `sysinfo::System`; `rss_kb(&mut self, pid) -> Option<u64>`; `fmt_kb(kb) -> String` ("1.2G"/"834M"). Cross-platform.
- `AppEvent::Tick` (timer thread ~10s). `App` fields: `mem_warn_kb: Option<u64>`, `reap_idle: bool`, `reap_timeout: Duration`, `idle_since: HashMap<String,Instant>`, `rss: HashMap<String,u64>`, `mem_sys: mem::Mem`.
- Idle tracking: hook→Idle inserts `idle_since[id]=now`; leaving Idle / focus / input removes it.
- Pure helper (unit-test): `fn should_park(state, focused, pinned, idle_for, timeout) -> bool` = `reap_idle && state==Idle && !focused && !pinned && idle_for>=timeout`. **Memory is NOT a park/kill trigger** — only a warning.
- On `Tick`: measure each live session's rss (store for UI). If `mem_warn` set and rss≥warn and this is the FIRST time it crossed (edge), fire `notify::desktop` "<label> is using <rss> — consider restarting" + mark it in the sidebar (warning marker). If `reap_idle`, park sessions where `should_park` (kill + `None` + `Parked` + clear idle_since). **No restart of active sessions anywhere.**
